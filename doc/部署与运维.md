# MindTrain 部署与运维

本文面向需要把 MindTrain 部署到本机、NAS 或云服务器的单用户。快速体验请先阅读仓库 [README](../README.md)，本文集中说明网络、安全、配置、备份、升级和排障。

## 1. 部署模型

```text
Internet / Local Network
          |
          v
   HTTPS Reverse Proxy
      |           |
      v           v
 MindTrain Web  Trainer MCP
      |           |
      +-----+-----+
            v
      Training Core
            |
            v
        PostgreSQL
```

- Web 和 Trainer MCP 是用户入口。
- Training Core 只需要被 Web 和 MCP 容器访问。
- PostgreSQL 不应暴露到公网。
- 当前部署按单实例、单用户设计，共享一个 Bootstrap Token。

## 2. Docker Compose 部署

### 2.1 准备环境

```bash
git clone https://github.com/shigella520/MindTrain.git
cd MindTrain/deploy/core-only
cp .env.example .env
```

至少修改：

```dotenv
POSTGRES_PASSWORD=replace-with-a-random-database-password
MINDTRAIN_BOOTSTRAP_TOKEN=replace-with-a-random-long-token
```

可使用以下命令分别生成随机值：

```bash
openssl rand -hex 32
```

### 2.2 启动与检查

```bash
docker compose up -d --build
docker compose ps
```

健康检查：

```bash
curl http://127.0.0.1:8080/actuator/health
curl http://127.0.0.1:8787/actuator/health
curl http://127.0.0.1:4173/healthz
```

默认端口：

| 服务 | 宿主机地址 | 说明 |
| --- | --- | --- |
| Web | `127.0.0.1:4173` | Dashboard、Web 训练和管理页面 |
| Trainer MCP | `127.0.0.1:8787` | Codex / MCP 客户端入口，业务请求需要 Token |
| Training Core | `127.0.0.1:8080` | REST API，通常无需公开 |
| PostgreSQL | 不映射 | 权威业务数据 |

数据库文件默认写入仓库根目录的 `var/postgres/`。不要将该目录提交到 Git，也不要在未备份时直接删除。

### 2.3 停止与重启

```bash
# 停止并保留数据库
docker compose down

# 重启全部服务
docker compose restart

# 只重启 MCP
docker compose restart trainer-mcp
```

不要使用 `docker compose down -v` 清理生产实例；虽然当前 Compose 使用宿主机目录而非命名卷，破坏性清理命令仍不适合作为日常操作。

## 3. 云端 HTTPS 与反向代理

推荐路由：

```text
https://mindtrain.example.com/     -> http://127.0.0.1:4173/
https://mindtrain.example.com/mcp -> http://127.0.0.1:8787/mcp
```

Web 容器会把同源 `/api/*` 请求转发给容器网络中的 Training Core，因此 Core 无需直接暴露公网。

### 3.1 Nginx 示例

```nginx
server {
    listen 443 ssl http2;
    server_name mindtrain.example.com;

    location = /mcp {
        proxy_pass http://127.0.0.1:8787/mcp;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Forwarded-Proto https;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header Authorization $http_authorization;
        proxy_buffering off;
        proxy_read_timeout 3600s;
    }

    location / {
        proxy_pass http://127.0.0.1:4173;
        proxy_set_header Host $host;
        proxy_set_header X-Forwarded-Proto https;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

证书签发、HTTP 到 HTTPS 跳转和通用安全响应头应按自己的网关规范配置。

### 3.2 安全边界

- 云端 Trainer MCP 必须使用 HTTPS。
- `/mcp` 业务请求要求 `Authorization: Bearer <MINDTRAIN_BOOTSTRAP_TOKEN>`。
- Core 和 PostgreSQL 不应直接公开到互联网。
- Token 不应出现在仓库、截图、日志、Issue 或反向代理配置正文中。
- 怀疑 Token 泄露时，修改 `.env` 后重建 Core 和 MCP，并重新配置客户端。

## 4. 启动环境变量

| 变量 | 默认值 | 用途 |
| --- | --- | --- |
| `POSTGRES_DB` | `mindtrain` | 数据库名 |
| `POSTGRES_USER` | `mindtrain` | 数据库用户 |
| `POSTGRES_PASSWORD` | 无安全默认值 | 数据库密码，必须修改 |
| `MINDTRAIN_BOOTSTRAP_USER_ID` | `local-admin` | 单用户管理员 ID |
| `MINDTRAIN_BOOTSTRAP_DISPLAY_NAME` | `Local Admin` | 管理员显示名 |
| `MINDTRAIN_BOOTSTRAP_TOKEN` | 无安全默认值 | Core 与 Trainer MCP 的访问令牌 |
| `MINDTRAIN_PUBLIC_DASHBOARD_ENABLED` | `false` | 是否匿名开放聚合看板接口 |
| `MINDTRAIN_CORE_PORT` | `8080` | Core 的 loopback 端口 |
| `MINDTRAIN_MCP_PORT` | `8787` | MCP 的 loopback 端口 |
| `MINDTRAIN_WEB_PORT` | `4173` | Web 的 loopback 端口 |

数据库连接、端口、Token 和公开访问开关属于基础设施或安全配置，因此保留为环境变量。

## 5. 数据库训练配置

训练策略保存在应用数据库中，可在 Web 的“管理 → 训练配置”修改，无需重启：

| 配置 | 默认值 | 说明 |
| --- | --- | --- |
| 每轮总题数 `questionCount` | `10` | 新 Session 的主问题目标 |
| 新题计划数 `newBudget` | `2` | 每轮最多引入的新题 |
| 复习题计划数 | `8` | 自动计算为 `questionCount - newBudget`，不单独存储 |
| 积压暂停阈值 | `20` | 到期题超过阈值时暂停新题 |
| 严重逾期天数 | `3` | 最老题超过该逾期天数时暂停新题 |
| AI 临时题有效期 | `24` 小时 | 过期未作答临时题会被物理清理 |
| 报表时区 | `Asia/Shanghai` | 决定“今日训练”的日期边界 |

## 6. 公开看板

默认情况下，报告和积压接口也需要 Token。仅在明确接受暴露聚合学习数据时设置：

```dotenv
MINDTRAIN_PUBLIC_DASHBOARD_ENABLED=true
```

匿名开放的接口仅包括：

- `GET /api/v1/reports/overview`
- `GET /api/v1/schedulers/backlog`

它们可能暴露作答数量、正确率、薄弱知识点和积压状态。创建会话、获取题目、提交答案、修改配置、导入和其他业务接口仍要求 Token。私人实例建议保持默认 `false`。

## 7. 备份与恢复

### 7.1 创建逻辑备份

在 `deploy/core-only` 目录执行：

```bash
mkdir -p ../../backup
docker compose exec -T postgres \
  sh -c 'pg_dump -U "$POSTGRES_USER" -d "$POSTGRES_DB" -Fc' \
  > ../../backup/mindtrain.dump
```

同时建议备份 `.env` 到密码管理器或受控的私密位置，不要提交到仓库。

### 7.2 恢复

先停止会写入数据的 Core 和 MCP，再恢复：

```bash
docker compose stop trainer-core trainer-mcp
docker compose exec -T postgres \
  sh -c 'pg_restore --clean --if-exists -U "$POSTGRES_USER" -d "$POSTGRES_DB"' \
  < ../../backup/mindtrain.dump
docker compose start trainer-core trainer-mcp
```

恢复属于破坏性操作，应先保留当前数据库备份，并优先在测试实例演练。

## 8. 升级

升级前先备份数据库，然后：

```bash
git pull --ff-only
cd deploy/core-only
docker compose up -d --build
docker compose ps
```

Training Core 启动时由 Flyway 自动执行数据库迁移。不要手工修改 Flyway 历史表或跳过失败迁移。

Codex Plugin 的升级流程见 [Codex Plugin 部署](CodexPlugin部署.md)。

## 9. 日志与故障排查

```bash
docker compose logs --tail=200 trainer-core
docker compose logs --tail=200 trainer-mcp
docker compose logs --tail=200 web
docker compose logs --tail=200 postgres
```

| 现象 | 优先检查 |
| --- | --- |
| Core 一直不健康 | PostgreSQL 健康状态、数据库密码、Flyway 错误 |
| MCP 返回 `401` | 客户端 Token 是否与 `.env` 一致 |
| Web 看板无数据 | 当前实例是否已有训练数据、浏览器实例设置、公开看板开关 |
| Codex 无法连接 | HTTPS、DNS、反向代理 `/mcp` 路由和 Token |
| 今日训练统计不对 | Web 管理页中的报表时区 |
| 修改训练配置未生效 | 新建 Session；已有 Session 保留创建时的题量目标 |

如果问题涉及 Plugin 加载、Marketplace 分支或本地配置，继续阅读 [Codex Plugin 部署](CodexPlugin部署.md)。
