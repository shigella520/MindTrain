# 知识目录

MindTrain 的知识目录由多个训练领域组成。每个领域拥有多个根知识点，根节点下可以继续形成任意层级的树；系统不存在唯一的全局根节点。

```text
知识目录
├── Java 后端
│   ├── JVM
│   └── 并发编程
└── Kubernetes
    ├── 工作负载
    └── 网络
```

## 初始化方式

领域与知识点只能由 Codex 发起初始化，正式数据由 Training Core 保存。支持两种来源：

- AI 对话：用户说明学习目标、受众、范围和难度，Codex 查询已有目录后规划知识点树。没有绑定参考资料时允许保存，但预览必须明确警告。
- 本地资料库：Codex 在本机索引用户指定目录，根据选定资料整理知识点树并绑定来源元数据。原文、绝对路径和本地索引不会上传 Core。

两种方式共用同一确认流程：

```text
Codex 生成草案 → Core 校验预览 → 用户查看完整树与差异 → 明确确认 → Core 原子保存
```

一份草案只针对一个新领域或已有领域，但可以包含多个根知识点。草案不可变；用户要求调整时，应废弃旧草案并重新预览。未确认、存在冲突或哈希不一致的草案不能进入正式目录。

## Codex 查询

MindTrain Plugin 可以列出领域、读取一个领域的完整知识点森林、搜索知识点以及查看节点详情。典型表达包括：

- “我目前有哪些训练领域？”
- “展示 Kubernetes 的知识点树。”
- “搜索 volatile 相关知识点。”
- “这个知识点有多少题，来源是什么？”

目录查询始终需要私人实例 Token，不受公开 Dashboard 配置影响。

## Web 目录

Web 的“知识目录”入口提供经过认证的只读管理视图：

- 领域列表与领域搜索。
- 多根知识点树、展开收起和关键词搜索。
- 知识点路径、说明、重要度、题目覆盖、掌握度、关键词和来源详情。
- 空目录时提供可复制的 Codex 初始化提示。

第一版不在 Web 内调用 AI，也不提供正式目录的编辑、移动、合并和删除操作。

## 数据边界

- Core 是 Domain、Topic、Relation、Source 和草案的唯一权威数据源。
- AI 对话是生成方式，不是权威资料来源；只有实际使用的本地或外部资料才进入 Source。
- Web、Trainer MCP 和 Skill 都不直接访问 PostgreSQL。
- 历史 Topic 会在数据库升级时补建对应 Domain，不修改题目、训练记录或掌握度。
